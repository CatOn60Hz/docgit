Version control for Word documents

Docgit

A Git-inspired workflow for Microsoft Word `.docx` files, with local snapshots, branching, diffs, rollback, and Word-aware file restoration.

Get Started How it works

Why It Exists

Standard Git works well for source code, but `.docx` files are zipped binary documents. Docgit stores document snapshots by hash and adds document-specific operations so versioning Word files feels practical instead of fragile.

Core Features

  • Commit and restore `.docx` snapshots
  • Branch-based document workflows
  • Status and commit history from the CLI
  • Paragraph-aware diffs with page and line hints
  • Safe merge behavior for diverged Word files

Quick Start

Initialize a repository in your working folder.

Commit your `.docx` file with a message.

Use branches, diffs, and rollback as the document evolves.

CLI Example

docgit init
docgit commit MyDocument.docx -m "Initial draft"
docgit status
docgit branch edits
docgit switch edits
docgit diff MyDocument.docx
docgit log

How It Works

Docgit creates a local `.docgit/` directory with an object store and an `index.json` file. Each commit maps tracked document paths to immutable file hashes, while branches act as lightweight pointers to commit IDs.

For diffs, Docgit extracts text from `.docx` files, annotates it with page and line context, and compares those lines rather than raw document bytes. For checkout and rollback, it can also try to close and reopen Word documents automatically through `pywin32`.
